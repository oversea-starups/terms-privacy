PhotoVault is a read-only export tool. It never deletes or edits your Photos originals.

Quick Checks

  1. Confirm Photos access is Full, or that the needed items are included in Limited Access.
  2. If an original is in iCloud, connect to an allowed network and keep PhotoVault open.
  3. Reconnect the same destination and use the system folder picker again if iOS asks.
  4. Confirm the destination is writable and has enough capacity plus the displayed reserve.
  5. Open the issue list, resolve the indicated condition, and retry eligible resources.

What “Verified” Means

For each verified resource, PhotoVault reread the destination file and matched its byte count and SHA-256 to the bytes Photos delivered during that export. This confirms the file at that moment; it does not guarantee future drive health or recreate every Photos-only feature.

Common Issues

  • Waiting for iCloud: confirm network access, keep the iPhone powered, and retry. PhotoVault does not substitute a lower-quality preview for an unavailable original.
  • Destination disconnected: reconnect the drive, cable, hub, and power; confirm the location appears in Files; then locate the same package.
  • Not enough space or read-only: free capacity or select a writable compatible destination. Previously verified files remain recorded.
  • Checksum mismatch: do not rely on that resource as verified. Retry it and, if the mismatch repeats, try another cable or drive while keeping the Photos original.
  • App closed or suspended: reopen the unfinished job, reconnect the same destination, and resume. The interrupted resource restarts from its file boundary.
  • Live Photo incomplete: retry the failed still-image or paired-video component.

Diagnostics

Open PhotoVault → Settings → Diagnostics. The default package excludes personal media, filenames, Photos identifiers, locations, album names, raw destination paths, and security-scoped bookmarks. It is never sent automatically; share it only if you choose.

快速检查

  1. 确认照片权限为“完全访问”，或所需项目已包含在“有限访问”范围中。
  2. 若原件位于 iCloud，请连接允许使用的网络，并保持 PhotoVault 在前台。
  3. 重新连接同一目标位置；若 iOS 要求，再次使用系统文件夹选择器定位。
  4. 确认目标位置可写，并有足够容量和界面显示的预留空间。
  5. 打开问题列表，解决提示条件后重试符合条件的资源。

“已校验”的含义

对于每个已校验资源，PhotoVault 会重新读取目标文件，并确认字节数和 SHA-256 与本次导出中“照片”提供的字节一致。这只确认当时的文件，不保证硬盘未来健康，也不保证重建“照片”独有的全部特性。

常见问题

  • 等待 iCloud：确认网络访问，保持 iPhone 供电并重试。PhotoVault 不会用低质量预览替代不可用原件。
  • 目标位置断开：重新连接硬盘、线缆、集线器和电源，在“文件”中确认目标可见，然后定位同一导出包。
  • 空间不足或目标只读：释放容量，或选择可写的兼容目标位置。先前已校验的文件仍会保留记录。
  • 校验值不一致：不要把该资源视为已校验。请重试；若再次发生，请尝试其他线缆或硬盘，并保留图库原件。
  • App 被关闭或挂起：重新打开未完成任务，连接同一目标位置并继续。中断资源会从文件边界重新开始。
  • 实况照片不完整：重试失败的静态图片或配对视频组成部分。

诊断

进入 PhotoVault → 设置 → 诊断。默认诊断包不包含个人媒体、文件名、Photos 标识、位置、相簿名、原始目标路径或安全作用域书签。它不会自动发送；只有在你选择时才会分享。
